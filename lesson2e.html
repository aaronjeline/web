1.2.1 Procedure for writing a method

I'd recommend using the following procedure for writing a method:

  1. Stub the method: Write the code that goes around the method, figure out what the return type should be and what the arguments should be.

    For example: let's say we were writing the isEven method, we'd first start by thinking:

    This method will take one input, which is an integer, and one output, which is a boolean.

    Then we'd write: 

static boolean isEven(int a){

}
  1. Write examples on paper.

1.2.2 More Boolean Operators

So far we've learned about and, or, and not. Here's the truth table for or:

Input A Input B Output
True True True
True False True
False True True
False False False

Here, I'll introduce a new boolean operator: nor. nor is short for "not or". The truth table is a follows:

Input A Input B Output
True True False
True False False
False True False
False False True

Write a method called nor that computes nor.

1.2.4 yepOrNope

Here's the yepOrNope method: 

static String yepOrNope(boolean b, int x) {
  if (b) {
    if (x > 5) {
      return "Yep!";
    } else {
      return "Nope!";
    }
  } else {
    return "Nope!";
  }
}

Now that you know about && (and) rewrite this method with only one if statement.

1.2.6 Some Math

Name these what you want.

  1. Write a method that takes two integers, and returns true if the first is the square root of the second.
  2. Write a method that determines if a number is positive.
  3. Write a method that takes a number and does the following:

1.2.7 The Collatz Conjecture

Write a method that does the following: Given a number named n: If the number is equal to 1, return "Done!" If the number is even: half it. If the number is odd: triple it, then add 3. Repeat until the number equals one.

As far as Mathmaticians know, this should always reach 1.
